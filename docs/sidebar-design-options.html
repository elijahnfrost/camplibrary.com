Sidebar tag system — three directions

Constant in all three: activity rows get their tinted grips back, spine bars come off the rows · events / QuickAdd / catalog keep the spine identity · everything below is clickable

The question each option answers differently: what does a tag look like, so that nothing else can be mistaken for one? Type, Where, Ages, Saved, and Kit all become ONE species per option.

① Pill family

Your instinct, executed strictly: every filter is the same compact pill toggle — Type pills fill with their category color, everything else fills accent green. Available kit is a pill too («Kit · 2 ⌄»), so even the long list enters the tag family. Two tidy lines for Type; nothing else wraps.

② Switch ledger

Maximum calm: every dimension is ONE ledger line — label left, control right. Where/Ages are mini segmented pills, Saved is a true toggle switch, Kit shows its count. Type collapses into a single «All types ⌄» control with a color swatch (click it to open the type menu). The whole filter block is five quiet lines.

③ Stamp sheet

Leans into the app's own hand-drawn voice: tags are rubber STAMPS — small rectangles that ink up (and sit slightly crooked) when pressed. Smaller than pills, so Type fits ONE line. Kit is a stamp with a count. Most character of the three; the crooked-when-selected detail is the camp-paper brand doing the work.
